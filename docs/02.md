# はじめに

前回は Qiskit を使って最初の量子回路を動かしました。

今回はいよいよ量子計算の核心：

* 重ね合わせ（superposition）
* 測定（measurement）
* 状態空間の指数的増加

を理解しながら、

> なぜ量子コンピュータは並列計算のように見えるのか？

を説明します。

ここが理解できると、量子アルゴリズムの見え方が一気に変わります。

---

# 1. 古典ビットと量子ビットの違い

まず復習です。

古典ビット：

```
0 または 1
```

量子ビット：

```
0 と 1 を同時に含める
```

例えば Hゲートを適用すると：

$$
|0\rangle \rightarrow \frac{|0\rangle + |1\rangle}{\sqrt{2}}
$$

これは

* 0でもあり
* 1でもある

状態です。

重要なのは：

> 2つの状態を同時に「保持」している

という点です。

---

# 2. 実際に重ね合わせを作ってみる

コードで確認します。

```python
from qiskit import QuantumCircuit

qc = QuantumCircuit(1, 1)

qc.h(0)

qc.measure(0, 0)

print(qc.draw("text"))
```

回路：

```
     ┌───┐┌─┐
q_0: ┤ H ├┤M├
     └───┘└╥┘
c: 1/══════╩═
```

意味：

| 操作 | 内容       |
| -- | -------- |
| H  | 重ね合わせを作る |
| M  | 測定       |

---

# 3. 測定するとどうなるか？

量子状態は測定すると確定します。

例えば：

```
(|0⟩ + |1⟩)/√2
```

は測定すると：

```
50% → 0
50% → 1
```

になります。

つまり：

> 測定するまで状態は確定していない

というのが量子計算の基本ルールです。

---

# 4. shotsとは何か？

次のコードを実行します：

```python
from qiskit import QuantumCircuit
from qiskit_aer import AerSimulator
from qiskit.compiler import transpile

qc = QuantumCircuit(1, 1)

qc.h(0)
qc.measure(0, 0)

sim = AerSimulator()

compiled = transpile(qc, sim)

result = sim.run(compiled, shots=1000).result()

print(result.get_counts())
```

例：

```
{'0': 492, '1': 508}
```

ここで：

```
shots=1000
```

とは：

```
回路を1000回実行する
```

という意味です。

量子計算は確率的なので、

1回ではなく統計で見る必要があります。

---

# 5. ここまでは「並列計算」ではない

ここが重要ポイントです。

よくある誤解：

```
量子ビットは0と1を同時に計算している
```

これは正しくありません。

正確には：

```
0と1を同時に含む1つの状態を作っている
```

だけです。

並列計算のように見えるのはまだ先です。

---

# 6. 並列らしさは2量子ビットから現れる

では2量子ビットを考えます。

初期状態：

```
|00⟩
```

ここにHゲートを2回適用すると：

```
(|00⟩ + |01⟩ + |10⟩ + |11⟩)/2
```

つまり：

```
4つの状態を同時に持つ
```

ことになります。

3量子ビットなら：

```
8状態
```

n量子ビットなら：

```
2^n状態
```

です。

ここが量子計算の核心です。

---

# 7. 古典コンピュータとの違い

古典コンピュータ：

```
000
001
010
011
100
101
110
111
```

1回の計算で扱えるのは1つだけ

量子コンピュータ：

```
全部まとめて1つの状態として扱う
```

つまり：

```
状態空間そのものを同時に操作できる
```

これが

> 並列計算のように見える理由

です。

---

# 8. でも答えが全部同時に出るわけではない

ここが最大の誤解ポイントです ⚠️

例えば：

```
f(x)
```

を量子計算すると：

```
f(00)
f(01)
f(10)
f(11)
```

全部取得できる？

答え：

```
できません
```

測定すると：

```
どれか1つだけ
```

しか観測できません。

---

# 9. ではなぜ高速になるのか？

ここが量子アルゴリズムの本質です。

量子計算は：

```
全部の候補を同時に試す
```

のではなく：

```
正解だけ観測されやすくする
```

という操作を行います。

これを：

```
量子干渉（interference）
```

と呼びます。

---

# 10. 量子計算のイメージ

古典探索：

```
1つずつ試す
```

量子探索：

```
全部を重ねる
↓
間違いを打ち消す
↓
正解だけ強調する
```

つまり：

並列計算ではなく

```
確率分布の操作
```

です。

---

今回学んだポイント：

✅ 量子計算は並列計算ではなく「確率の操作」

そして：

> 並列計算のように見える理由は
> 状態空間を同時に操作できるから

です。

# 測定とは何をしているのか？

まず結論から言うと：

量子ビットは測定するまで確定した値を持っていません。

たとえば次の状態：

```
(|0⟩ + |1⟩)/√2
```

これは

* 0でもあり
* 1でもある

状態です。

しかし測定すると：

```
0 か 1 のどちらか
```

になります。

しかも確率は：

```
50% : 50%
```

です。

---

# なぜ観測すると状態が壊れるのか？

量子ビットは測定前：

```
重ね合わせ状態
```

にあります。

しかし測定すると：

```
|0⟩ または |1⟩
```

のどちらかに確定します。

これを：

**波動関数の収縮（wave function collapse）**

と呼びます。

つまり：

測定 = 情報を取り出す操作

ではなく：

測定 = 状態を変えてしまう操作

です。

ここが古典ビットとの最大の違いです。

---

# 実際に回路で確認してみる

前回と同じように、Hゲートを使ってみます。

```python
from qiskit import QuantumCircuit

qc = QuantumCircuit(1, 1)

qc.h(0)
qc.measure(0, 0)

qc.draw("text")
```

意味：

```
|0⟩
 ↓
H
 ↓
(|0⟩ + |1⟩)/√2
 ↓
measure
 ↓
0 or 1
```

つまり：

測定すると重ね合わせは消えます。

# 確率はどこから生まれるのか？

ここが重要です。

量子コンピュータの確率は：

ノイズではありません

誤差でもありません

仕様です。

たとえば：

```
(|0⟩ + |1⟩)/√2
```

なら：

|0⟩ → 50%
|1⟩ → 50%

になります。

別の状態：

```
√0.8|0⟩ + √0.2|1⟩
```

なら：

|0⟩ → 80%
|1⟩ → 20%

になります。

つまり：

振幅の2乗 = 出現確率

です。

---

# なぜ量子回路は何度も実行するのか？

ここが実践で最初に戸惑うポイントです。

古典プログラム：

```
1回実行 → 1つの結果
```

量子プログラム：

```
1回実行 → ランダムな結果
```

です。

そのため：

100回
1000回
10000回

と繰り返して実行します。

例：

```python
from qiskit import Aer, execute

backend = Aer.get_backend("qasm_simulator")

job = execute(qc, backend, shots=1000)

result = job.result()

print(result.get_counts())
```

出力例：

```
{'0': 503, '1': 497}
```

これは：

確率分布を観測している

という意味です。

---

# なぜ確率分布が重要なのか？

量子アルゴリズムは：

正しい答えを直接出す

のではありません。

代わりに：

正しい答えが出やすい状態を作る

のです。

つまり：

古典計算：

```
計算 → 答え
```

量子計算：

```
状態生成 → 測定 → 確率的に答え
```

です。

量子アルゴリズムとは：

確率の設計

なのです。

---

# Bell状態を測定すると何が起きるか？

前回作ったBell状態：

```
(|00⟩ + |11⟩)/√2
```

これを測定すると：

```
00
または
11
```

が出ます。

しかも：

```
01 は出ない
10 も出ない
```

です。

つまり：

片方を測定するともう片方も決まる

これが量子もつれです。

---

# ここまでの理解が量子アルゴリズムの入口

ここまでで重要なのは：

量子計算は値を計算しているのではない

ということです。

代わりに：

状態を設計している

のです。

そして：

測定によって確率として答えが現れる

のです。

これが：

* Grover探索
* Shorの素因数分解
* 量子機械学習

すべての基礎になります。

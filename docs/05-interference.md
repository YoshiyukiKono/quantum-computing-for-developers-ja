# 干渉とは何か？

量子ビットは単なる確率ではありません。

重要なのは：

**確率の前に「振幅（amplitude）」がある**

ということです。

古典ビット：

```
0 or 1
```

確率ビット：

```
0 が 70%
1 が 30%
```

量子ビット：

```
|ψ⟩ = α|0⟩ + β|1⟩
```

ここで：

* α, β は確率ではない
* 振幅（amplitude）
* 観測すると |α|², |β|² が確率になる

つまり：

振幅は直接観測できないが操作できる

これがすべての鍵です。🔑

---

# 干渉とは「振幅の足し算」

量子状態は重ね合わせでした：

```
(|0⟩ + |1⟩)/√2
```

ここにもう一度 Hadamard をかけるとどうなるでしょう？

```
(|0⟩ + |1⟩)/√2
    ↓
|0⟩
```

なぜでしょう？

答え：

**振幅が足し合わさるから**

です。

具体的には：

```
+1/√2 + +1/√2 → 強め合う
```

逆に：

```
+1/√2 + -1/√2 → 打ち消し合う
```

これが干渉です。

---

# 波としての量子状態

量子状態は粒ではなく：

**波**

として振る舞います。

波には特徴があります：

* 強め合う
* 打ち消し合う

つまり：

```
正しい答え → 強める
間違った答え → 消す
```

という操作が可能になります。

これが量子アルゴリズムの本質です。🚀

---

# なぜ確率を増やせるのか？

古典コンピュータ：

```
確率は操作できない
```

量子コンピュータ：

```
振幅を操作できる
```

そして：

```
確率 = 振幅²
```

つまり：

振幅を大きくする
→ 確率が上がる

となります。

例：

```
振幅 1/2 → 確率 1/4
振幅 1   → 確率 1
```

だから：

確率を増やせる

のです。

---

# なぜ間違った答えを消せるのか？

量子状態はこう書けます：

```
|ψ⟩ =
正解の振幅 × |correct⟩
+
不正解の振幅 × |wrong⟩
```

ここで：

不正解の振幅を打ち消す

ことができます。

つまり：

```
|wrong⟩ の振幅 → 0
```

にできる。

結果：

```
|correct⟩ だけ残る
```

これが量子探索アルゴリズム（Grover）の仕組みです。

量子アルゴリズムの本質は：

**干渉の設計**

1. 全候補を重ね合わせで用意する

これを繰り返します。

結果：

```
観測すると正解が出る
```

確率が上がります。

# Pythonで干渉を確認する

ここでは Qiskit を使って、干渉を実際に確認します。

まず、1量子ビットに Hadamard ゲートを1回だけかけて測定します。

```python
from qiskit import QuantumCircuit
from qiskit_aer import AerSimulator

qc = QuantumCircuit(1, 1)

qc.h(0)
qc.measure(0, 0)

sim = AerSimulator()
result = sim.run(qc, shots=1000).result()
counts = result.get_counts()

print(counts)
print(qc.draw("text"))
```

実行結果は、おおよそ次のようになります。

```text
{'0': 502, '1': 498}
```

これは、`|0⟩` に Hadamard をかけることで、

```text
(|0⟩ + |1⟩) / √2
```

という重ね合わせが作られたためです。

測定すると、`0` と `1` がほぼ半々で出ます。

---

# Hadamardを2回かける

次に、Hadamard を2回かけます。

```python
from qiskit import QuantumCircuit
from qiskit_aer import AerSimulator

qc = QuantumCircuit(1, 1)

qc.h(0)
qc.h(0)
qc.measure(0, 0)

sim = AerSimulator()
result = sim.run(qc, shots=1000).result()
counts = result.get_counts()

print(counts)
print(qc.draw("text"))
```

実行結果は次のようになります。

```text
{'0': 1000}
```

不思議に見えます。

1回目の Hadamard で `0` と `1` が半々になるなら、2回目も半々になりそうです。

しかし、実際には `0` だけが出ます。

これが **干渉** です。

---

# 何が起きているのか？

Hadamard ゲートは、次のような変換をします。

```text
|0⟩ → (|0⟩ + |1⟩) / √2
|1⟩ → (|0⟩ - |1⟩) / √2
```

1回目：

```text
|0⟩
→ (|0⟩ + |1⟩) / √2
```

2回目：

```text
(|0⟩ + |1⟩) / √2
→ |0⟩
```

ここで重要なのは、`|1⟩` 側の振幅が打ち消し合うことです。

```text
+1/2 - 1/2 = 0
```

一方で、`|0⟩` 側の振幅は強め合います。

```text
+1/2 + 1/2 = 1
```

つまり：

```text
|0⟩ の振幅 → 1
|1⟩ の振幅 → 0
```

だから測定すると、必ず `0` が出ます。

---

# これは乱数ではない

ここが大事です。

Hadamard を1回だけかけると、結果はランダムに見えます。

```text
0 と 1 が半々
```

しかし、Hadamard を2回かけると、必ず元に戻ります。

```text
0 だけが出る
```

つまり量子状態は、単なるランダムではありません。

測定する前には、振幅という情報があり、その振幅同士が干渉します。

量子コンピュータは、この干渉を使って、

```text
正しい答えの振幅を強める
間違った答えの振幅を弱める
```

という計算をします。

# コラム①：Hadamard を2回かけると元に戻るということの意味

**Hadamard を2回かけると元に戻る（ (H^2 = I) ）** という性質は、量子アルゴリズムの中で実はかなり重要な役割を持っています。

ここでは代表的な「使い道」を開発者視点で整理します。

---

# ① 重ね合わせを作ってから回収する（基本パターン）

Hadamard の最も基本的な使い方は：

```
H → 位相操作 → H
```

です。

流れとしては：

1. H で状態を広げる
2. 途中で情報を書き込む（位相操作）
3. H で情報を読み出す

という構造になります。

これは量子アルゴリズムの基本テンプレートです。

---

# ② 位相を確率に変換する（超重要）

量子計算では：

* 位相（phase）
* 確率（measurement probability）

は別物です

でも Hadamard を使うと：

> 位相の違いを観測できる確率に変換できる

ようになります。

例：

```
|0⟩+|1⟩
|0⟩−|1⟩
```

は測定すると区別できませんが、

もう一度 H をかけると：

```
|0⟩
|1⟩
```

になって区別可能になります。

つまり：

Hadamard = 位相を読み出す装置

です。

「干渉を設計する」ための装置

# コラム③：ψとは？

量子コンピューティングで出てくる **ψ（プサイ）** は、

> 量子状態（quantum state）そのもの

を表す記号です。

つまり：

```text
ψ = その量子ビットが「どんな状態にあるか」の全部の情報
```

をまとめて表しています。

---

# ψの正体（直感）

古典ビットなら：

```text
0 か 1
```

だけです。

でも量子ビットは：

```text
0 と 1 の重ね合わせ
```

になります。

それをまとめて書くと：

```text
ψ = a|0⟩ + b|1⟩
```

です。

ここで：

* a = |0⟩になる強さ（係数）
* b = |1⟩になる強さ（係数）

です。

そして：

```text
|a|² + |b|² = 1
```

が必ず成立します（確率になるため）。

---

# なぜ ψ を使うのか？

理由は：

状態が 0 でも 1 でもないから

です。

例えば：

```text
ψ = (|0⟩ + |1⟩)/√2
```

これは

```text
50%で0
50%で1
```

という意味になります。

これはまさに Hadamard を1回かけた状態ですね。

---

# Hadamardとの関係

あなたが先ほど考えていた：

> H を2回かけると元に戻る

という話は、

実はこう書けます：

```text
H(Hψ) = ψ
```

つまり：

Hadamard は ψ を変換する操作

です。

---

# ψは「波」でもある

量子力学では ψ は

> 波動関数（wavefunction）

とも呼ばれます。

重要なのは：

* 大きさ → 確率になる
* 符号（＋／−）
* 位相（複素数）

すべてが意味を持つ

という点です。

特に：

```text
|0⟩ + |1⟩
```

と

```text
|0⟩ − |1⟩
```

は確率は同じですが、

**干渉の結果が変わる**

ので別の ψ です。

ここに量子アルゴリズムの核心があります✨

---

# 開発者向けの理解まとめ

ψ はこう考えると分かりやすいです：

| 記号 | 意味 |
|------|------|
| ψ | 量子状態そのもの |
| a | \|0⟩ の係数（振幅） |
| b | \|1⟩ の係数（振幅） |
| \|a\|² | 0 が観測される確率 |
| \|b\|² | 1 が観測される確率 |

つまり：

> ψ = 量子ビットの「完全な内部状態」

です。

Hadamard や CNOT などの量子ゲートは、すべて ψ を変換しています。

# 1. SAT問題とは何か？

SAT（Boolean Satisfiability Problem）は：

> 条件を満たす入力が存在するか？

を調べる問題です。

例：

```
(x1 OR x2) AND (NOT x1 OR x3)
```

満たす組み合わせを探します。

例えば：

```
x1 = 0
x2 = 1
x3 = 1
```

なら成立します。

つまりSATとは：

```
条件を満たすビット列を探す問題
```

です。

これはGroverと非常に相性が良い問題です。

理由：

Groverは

> 正解を探索するアルゴリズム

だからです。

---

# 2. SAT問題を量子回路として書く考え方

古典計算：

```
if condition(x):
    success
```

量子計算：

```
condition(x) == True のとき位相反転
```

つまり：

SAT問題の条件式そのものが

**オラクル**

になります。

設計手順：

① 条件式を書く
② AND / OR / NOT に分解
③ 多制御ゲートへ変換
④ 位相反転にする

これだけです。

---

# 3. 例題：簡単なSAT問題

次を解いてみます：

```
(x1 AND x2) = True
```

```

```

だけです。

つまりオラクルは：

```
|11⟩だけ位相反転
```

です。

これは：

```
CZゲート
```

で実装できます。

# 6. 可逆回路に変換する理由

量子回路では：

```
x AND y
```

だけを書くことはできません。

理由：

量子計算は

> 可逆（reversible）

だからです。

例えば：

古典：

```
1 AND 0 = 0
0 AND 0 = 0
```

区別できません。

つまり：

逆変換できません。

量子回路では：

```
(x, y, 0) → (x, y, x AND y)
```

のように書きます。

これを：

Toffoliゲート

と呼びます。

---

# 7. SATオラクルの設計テンプレート

SATオラクルは通常こう書きます：

```
入力レジスタ
補助レジスタ
出力レジスタ
```

流れ：

```
条件計算
↓
結果を出力ビットへ保存
↓
位相反転
↓
計算を元に戻す
```

最後の：

```
元に戻す
```

が重要です。

これを：

**uncompute**

と呼びます。

---

# 8. なぜuncomputeが必要か？

例えば：

途中計算を消さないと：

```
|x⟩ → |x, garbage⟩
```

になります。

すると：

干渉が壊れます。

だから：

```
|x⟩ → 位相反転 → |x⟩
```

に戻す必要があります。

つまり：

量子回路設計では：

```
計算する
戻す
```

がセットになります。

# 10. QiskitでSATオラクルを書く

例：

```
x1 AND x2
```

を解くGrover回路：

```python
from qiskit import QuantumCircuit

qc = QuantumCircuit(3)

# 重ね合わせ
qc.h([0,1])

# AND計算
qc.ccx(0,1,2)

# 位相反転
qc.z(2)

# uncompute
qc.ccx(0,1,2)

qc.draw("text")
```

構造：

```
計算
↓
位相反転
↓
戻す
```

になっています。

これが：

SATオラクルの基本形です。

# 11. 回路サイズはどう増えるか？

SAT問題は一般に：

```
NP完全
```

です。

つまり：

条件が増えると：

```

Toffoliが増える
深さが増える
```

します。

例えば：

変数数：

```

```

```

```

なら：

回路サイズは：

```
O(m)
```

程度で増えます。

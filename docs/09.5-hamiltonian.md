今回は：

**量子アルゴリズムの共通言語**

ハミルトニアン

について解説します。

---

# 1. ハミルトニアンとは何か？

量子コンピューティングで頻出する式：

```
H |ψ⟩ = E |ψ⟩
```

ここで：

* H：ハミルトニアン
* |ψ⟩：状態
* E：エネルギー

つまり：

> 状態のエネルギーを計算する演算子

です。

古典計算で言えば：

```python
energy(state)
```

のような関数に相当します。

---

# 2. もともとは物理の概念

ハミルトニアンは元々：

物理学の概念です。

例えば古典力学では：

```
H = 運動エネルギー + 位置エネルギー
```

でした。

量子力学では：

```
H = 系のエネルギーを測る演算子
```

になります。

つまり：

> 系の振る舞いを決める最も重要な式

です。

---

# 3. なぜ量子アルゴリズムで重要なのか？

量子アルゴリズムでは：

問題

↓

ハミルトニアン

に変換します。

例えば：

| 問題      | ハミルトニアン |
| ------- | ------- |
| 分子構造    | 電子相互作用  |
| Max-Cut | 辺の切断数   |
| SAT     | 制約違反数   |
| 最短経路    | 距離の総和   |

つまり：

```
問題 = エネルギー最小化問題
```

として書き直します。

---

# 4. なぜエネルギー最小化になるのか？

自然界では：

系は常に

**最小エネルギー状態**

へ向かいます。

例えば：

* 水は低い場所に流れる
* バネは自然長に戻る
* 電子は安定軌道に落ちる

同じ考え方を使って：

```
最適解 = 最小エネルギー
```

として問題を書き換えます。

---

# 5. ハミルトニアンは「問題そのもの」

量子アルゴリズムでは：

回路を書く前に

まず：

```
H を書く
```

必要があります。

例えば：

Max-Cut問題なら：

```
H = Σ (1 − Zi Zj)/2
```

になります。

SAT問題なら：

```
制約違反ごとに +1
```

のような形になります。

つまり：

> ハミルトニアンは問題の設計図

です。

---

# 6. なぜNISQ時代はハミルトニアン中心なのか？

第9回で説明した通り、

現在の量子コンピュータには：

* 回路深さ制限
* ノイズ
* 量子ビット不足

があります。

そこで使われるのが：

```
ハミルトニアン最小化型アルゴリズム
```

です。

代表例：

* VQE
* QAOA
* 量子アニーリング

つまり：

現代の量子計算は

```
回路設計
↓
ではなく
↓
ハミルトニアン設計
```

になっています。

---

# 7. ハミルトニアンはどう書くのか？

実際には：

パウリ演算子の和

として書きます。

例：

```
H = Z₀Z₁ + X₀X₁ + Z₀
```

これは：

3つの測定の和です：

```
⟨Z₀Z₁⟩
+
⟨X₀X₁⟩
+
⟨Z₀⟩
```

つまり：

量子回路で測定できます。

---

# 8. なぜこの形にするのか？

理由はシンプルです：

パウリ演算子は

量子コンピュータで直接測定できる

からです。

例えば：

| 演算子 | 測定方法        |
| --- | ----------- |
| Z   | そのまま測定      |
| X   | Hの後に測定      |
| Y   | S† → H → 測定 |

つまり：

```
測定可能な形
=
パウリ演算子分解
```

です。

---

# 9. ハミルトニアンと量子アルゴリズムの関係

量子アルゴリズムの多くは：

次の形になります：

```
H を作る
↓
最小固有値を探す
```

代表例：

| アルゴリズム | 目的     |
| ------ | ------ |
| VQE    | 基底状態探索 |
| QAOA   | 最適解探索  |
| QPE    | 固有値計算  |

つまり：

> 量子アルゴリズムとは固有値問題を解く装置

とも言えます。

---

量子アルゴリズム：


つまり：

ハミルトニアンは

**量子プログラムの仕様書**

です。

テンソル積（**tensor product**）は、量子コンピューティングで最初の大きな壁の一つです。特に

* 複数量子ビット
* エンタングルメント
* 多次元状態空間
  を理解する鍵になります。

ここでは**開発者目線の「とっかかり」**として説明します。🧭

---

# 1. まず直感：テンソル積は「状態を並べる操作」

古典ビットなら：

```
bit A = 0
bit B = 1
```

まとめると：

```
(0,1)
```

量子では：

```
|0⟩ ⊗ |1⟩
```

と書きます。

つまり：

> テンソル積 = 「別々の状態空間を合体する操作」

です。

---

# 2. なぜ普通のベクトル結合ではダメなのか？

古典なら：

```
[0]
[1]
```

を並べればOKです。

でも量子では：

```
a|0⟩ + b|1⟩
```

のように**重ね合わせ**があります。

さらに2量子ビットになると：

```
a|00⟩ + b|01⟩ + c|10⟩ + d|11⟩
```

になります。

ここで重要：

**状態の数が掛け算で増える**

```
2 × 2 = 4
```

これを作るのがテンソル積です。

---

# 3. 実際に計算してみる

例：

```
|0⟩ = [1
       0]

|1⟩ = [0
       1]
```

すると：

```
|0⟩ ⊗ |1⟩
```

は：

```
[1
 0] ⊗ [0
       1]
=
[0
 1
 0
 0]
```

になります。

つまり：

> ベクトルの次元が増える操作

です。

# 4. プログラマ向け理解（重要）

テンソル積は：

```
配列のネスト
```

に近いです。

例えば：

```python
A = [a, b]
B = [c, d]
```

なら：

```
A ⊗ B =
[a*c,
 a*d,
 b*c,
 b*d]
```

です。

つまり：

**全組み合わせを作る**

操作です。

# 5. なぜ量子では必須なのか？

理由は：

量子ビットは独立に存在しないからです。

例えば：

```
|ψ⟩ = |0⟩ ⊗ |1⟩
```

は分離可能状態ですが：

```
|ψ⟩ = (|00⟩ + |11⟩)/√2
```

は分離できません。

これが：

エンタングルメント

です。

つまり：

> テンソル積を理解しないと量子もつれが理解できない

# 1. 分離可能状態とは何か

まずこの状態：

[
|0⟩ \otimes |1⟩
]

これは「1番目の量子ビットが0」「2番目が1」です。

つまり：

> 個別の量子ビットの状態を書き並べただけ

です。

例えば：

[
(a|0⟩ + b|1⟩) \otimes |0⟩
]

なら展開すると：

[
a|00⟩ + b|10⟩
]

これはまだ「分離可能」です。

なぜなら：

[
(\text{1量子ビット}) \otimes (\text{1量子ビット})
]

の形に戻せるからです。

---

# 2. 分離できない状態が現れる

ここで重要な例：

[
(|00⟩ + |11⟩)/\sqrt{2}
]

これは次のようには書けません：

[
(\alpha|0⟩+\beta|1⟩)\otimes(\gamma|0⟩+\delta|1⟩)
]

つまり：

> テンソル積に分解できない

これが：

✨ エンタングルメント（量子もつれ）

です。

---

# 3. なぜこれが重要なのか（開発者視点）

普通のデータ構造なら：

```python
state = (bitA, bitB)
```

のように分解できます。

しかし量子では：

```python
state != (qubitA, qubitB)
```

になることがあります。

つまり：

状態が「ペア」ではなく「全体」になる

これが量子計算の本質です。

# 4. テンソル積は「状態空間を作る操作」

ここが核心です。

1量子ビット：

[
\mathbb{C}^2

2量子ビット：

[
\mathbb{C}^2 \otimes \mathbb{C}^2 = \mathbb{C}^4
]

3量子ビット：

[
\mathbb{C}^8

つまり：

> 量子ビットを増やす = 空間の次元が指数的に増える 📈

これが「量子並列性」の正体です。

つまり：

* X → 1番目に作用

になります。

# 1. まずCNOTとは何か（古典版）

CNOT は：

```
control = 1 のときだけ
target を反転
```

つまり：

| control | target | 出力 |
| ------- | ------ | -- |
| 0       | 0      | 00 |
| 0       | 1      | 01 |
| 1       | 0      | 11 |
| 1       | 1      | 10 |

古典なら普通の if 文です：

```python
if control == 1:
    target = not target
```

---

# 2. 量子版CNOTの違い

量子では control が：

```
0 と 1 の重ね合わせ
```

になれます。

例えば：

[
|+\rangle = (|0\rangle + |1\rangle)/\sqrt{2}
]

ここに CNOT をかけます。

初期状態：

[
|+\rangle \otimes |0\rangle
]

展開すると：

[
(|0\rangle + |1\rangle)/\sqrt{2} \otimes |0\rangle
]

つまり：

[
(|00\rangle + |10\rangle)/\sqrt{2}
]

---

# 3. ここでCNOTを適用する

CNOTは：

```
|10⟩ → |11⟩
```

なので：

[
(|00\rangle + |10\rangle)/\sqrt{2}
]

は：

[
(|00\rangle + |11\rangle)/\sqrt{2}
]

になります。

これが：

✨ Bell状態

です。

---

# 4. なぜこれが特別なのか？

この状態：

[
(|00\rangle + |11\rangle)/\sqrt{2}
]

は次の形にできません：

[
|\psi\rangle \otimes |\phi\rangle
]

つまり：

テンソル積に分解できない

＝エンタングルメント

です。

---

# 5. 何が起きているのか（直感）

処理の流れ：

① Hadamard
② CNOT

だけです：

```
|0⟩ → H → |+⟩
```

次に：

```
|+⟩|0⟩ → CNOT → Bell状態
```

つまり：

> 重ね合わせ × 条件付き操作
> = エンタングルメント

です。

---

# 6. なぜCNOTだけが特別なのか？

重要な事実：

単一量子ビットゲートだけでは

エンタングルメントは作れません

例えば：

```
H ⊗ I
X ⊗ I
Z ⊗ I
```

などは全部：

分離可能状態しか作れない

しかし：

```
CNOT
```

は：

量子ビット同士を結びつける

だから：

回路設計の最小単位は

```
single qubit gate
+
CNOT
```

になります。

これは実は：

量子回路の基本定理

です。

# 1. Hadamard の正体

Hadamard ゲートは次の変換です：

H = \frac{1}{\sqrt{2}} \begin{bmatrix}1 & 1 \ 1 & -1\end{bmatrix}

作用：

[
H|0\rangle = \frac{|0\rangle + |1\rangle}{\sqrt{2}}
]

[
H|1\rangle = \frac{|0\rangle - |1\rangle}{\sqrt{2}}
]

ここで重要なのは：

最後の **−（マイナス）**

です。

# 2. なぜ 50% / 50% が本質ではないのか？

多くの説明では：

```text

     50%で1
```

と説明されます。

しかしこれは：

測定した後の話

です。

量子状態としては：

[
|0\rangle + |1\rangle
]

ではなく：

[
|0\rangle - |1\rangle
]

も存在します。

つまり：

> Hadamard は「確率」を作るのではなく「符号付き重ね合わせ」を作る

のです。

[

途中で

プラスとマイナスが打ち消し合う

からです。

# 4. 干渉とは何か（核心）

Hadamard を2回かけると：

```text
+ と +
+ と −
```

が混ざります。

すると：

```text
強め合う
弱め合う
```

が発生します。

これが：

干渉（interference）

です。

つまり：

> 量子計算は「確率を操作する」のではなく「振幅を干渉させる」

のです。

# 1. まず「回路の深さ」とは何か

量子回路の深さ（depth）とは：

```text
ゲートの層の数
```

です。

例：

```
q0 ──H──■────
         │
q1 ─────X──■─
            │
q2 ─────────X
```

これは：

depth = 3

くらいの回路です。

つまり：

> 時間方向の長さ

です。

---

# 2. なぜ深さが重要なのか

理由はシンプル：

```text
エンタングルメントは伝播する
```

からです。

例えば：

1層：

```
q0 ──■──
     │
q1 ──X──
```

ここでは：

q0 と q1 だけ絡む

---

# 3. 2層になるとどうなるか

```
q0 ──■────
     │
q1 ──X──■─
         │
q2 ──────X
```

すると：

q0 と q2 も間接的に絡む

---

# 4. 深くなると何が起きるか

層が増えると：

```
q0 ↔ q1 ↔ q2 ↔ q3 ↔ q4 ...
```

のように：

絡みが広がります。

つまり：

```text
局所エンタングルメント
→
全体エンタングルメント
```

になります。

---

# 5. tensor network が効く条件

tensor network は：

こういう状態に強い：

```
q0 ↔ q1

q2 ↔ q3
```

つまり：

局所的な結合

です。

しかし：

```
q0 ↔ q1 ↔ q2 ↔ q3 ↔ q4 ↔ ...
```

になると：

分解できなくなります。

---

# 6. 回路の深さとエンタングルメントの関係

重要な法則：

```text
最大エンタングルメント量 ≈ 回路深さ
```

です。

つまり：

浅い回路：

```
低エンタングルメント
```

深い回路：

```
高エンタングルメント
```

になります。

---

# 7. だから浅い回路はシミュレートできる

例えば：

depth = 2

なら：

```
影響範囲 ≈ 2量子ビット先まで
```

です。

つまり：

巨大テンソルにならない

ので：

古典計算で処理できます。

---

# 8. 深い回路が突然難しくなる理由

depth が増えると：

```
影響範囲 ≈ 全量子ビット
```

になります。

すると：

状態が：

```
rank-n tensor
```

になります。

つまり：

指数爆発です。

---

# 9. これが NISQ アルゴリズムの設計思想

VQE：

```
浅い回路
+
古典最適化
```

QAOA：

```
浅い回路
+
パラメータ調整
```

つまり：

```text
エンタングルメントを制御する
```

設計です。

---

# 10. なぜ浅い回路が重要なのか（物理的理由）

現在の量子デバイス：

```
ノイズが多い
```

です。

深い回路：

```
エラー蓄積
```

浅い回路：

```
実行可能
```

つまり：

現実の制約でも浅い回路が必要です。

---

Google のランダム回路：

```
深い
+
広い
+
ランダム
```

つまり：

```
最大エンタングルメント
```

を作りました。

だから：

古典シミュレーション不能になりました。

qubit数 × depth × entanglement

になります。
